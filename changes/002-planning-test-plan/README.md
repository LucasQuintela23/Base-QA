# 002 - Planning Test Plan

## Goal
Define tactical QA scope and test strategy.

## Build
- In-scope / Out-of-scope
- Test approach (API, E2E, contract, etc.)
- Entry criteria (Ready)
- Exit criteria (Done)

## Checklist
- [ ] Scope boundaries are clear.
- [ ] Test types are mapped to business risks.
- [ ] Ready/Done criteria are measurable.

## Output
- docs/test_plan.md
